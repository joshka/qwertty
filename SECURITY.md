# Security Policy

## Supported Versions

Security fixes target the latest released qwertty version and `main`. Older releases may receive
fixes when the impact and upgrade path justify the extra maintenance.

## Reporting a Vulnerability

Please do not report suspected vulnerabilities in public issues.

Use GitHub private vulnerability reporting if it is available on the repository. Otherwise, contact
the maintainer through the email address on the maintainer's GitHub or crates.io profile.

Include enough detail to reproduce and assess the issue:

- affected qwertty version or commit;
- platform, terminal, and shell;
- the input bytes, command sequence, or policy configuration involved;
- expected impact;
- whether the issue requires untrusted terminal input, a malicious control sequence, or local
  access.

The maintainer will acknowledge actionable reports, investigate the impact, and coordinate a fix and
release when the issue is valid.
