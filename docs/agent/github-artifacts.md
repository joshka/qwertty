# GitHub Artifacts

GitHub artifacts should make the project evolution visible without creating a large stale backlog.

## Issues

Use issues for PR-sized slices. Each issue should include the problem, scope, out-of-scope work,
behavior references, and acceptance criteria.

## Pull Requests

Every pull request should explain:

- summary;
- design notes;
- user-facing surface;
- tests or validation;
- reference material used;
- follow-up.

## ADRs

Use ADRs for durable decisions, especially package boundaries, terminal ownership, parser
architecture, query routing, policy, registry purpose, and release scope changes.
